# 一次真实清理：为什么释放了 12.49 GB，却没有乱删

这是一次已经脱敏的 Windows C/E 盘清理案例。它展示的是判断方法，不是对每台电脑的空间承诺。

## 结果

| 清理对象 | 释放空间 | 为什么可以删 |
|---|---:|---|
| 公开安装包、APK、未完成下载 | 9.98 GB | 软件可从官网或应用商店重新下载，不影响已安装的软件 |
| SHA-256 完全相同的文件副本 | 1.16 GB | 每组保留一份完整原件，删除其余字节完全相同的副本 |
| 已验证的旧资料副本 | 1.36 GB | 有独立保留位置，且有效文件已被覆盖 |
| **合计** | **12.49 GB** | 先证明可恢复、可替代或已有副本，再删除 |

## 没有碰什么

- WSL 和 Docker 虚拟磁盘
- Claude/Codex 正在使用的运行环境
- 当前 Windows 桌面
- 聊天软件资料
- 未经验证的 PDF、视频、项目文件和唯一文档

## 这次案例教会了什么

“目录名字像桌面”不等于“它就是当前 Windows 桌面”。迁移目录、历史副本和当前系统桌面必须分别核验：

1. 查询 Windows 当前实际桌面路径和目录联接。
2. 比较候选副本与保留副本的相对路径、大小和文件数量。
3. 执行前再检查一次；Office 临时文件或正在变化的内容会让原来的判断失效。

这条规则已经写入 Skill：迁移目录只能作为比较对象，不能因为名字相似就自动删除。

## 核心原则

> 删除后的真实恢复成本，低于它持续占用硬盘的成本，就建议删除。

公开安装包的恢复成本低；唯一资料和运行环境的恢复成本高。安全不是“不删除”，而是先证明可恢复后果断删除。
